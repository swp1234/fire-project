## 🔧 기술적 상세

### 렌더링 차단(Render-Blocking) 방지

**문제:**
```html
<!-- 이전: HTML 파싱 중단, 스크립트 로드 대기 -->
<script src="js/app.js"></script>
```

**해결:**
```html
<!-- 개선: HTML 파싱 계속, 백그라운드 로드, DOM 준비 후 실행 -->
<link rel="preload" href="js/app.js" as="script">
<script src="js/app.js" defer></script>
```

### 레이아웃 스래싱(Layout Thrashing) 방지

**문제:**
```css
/* 매 프레임 16ms마다 이 속성들이 변함 */
transition: all 0.3s;
/* ↓ 변하는 속성 예시 ↓ */
width, height, padding, margin, left, top, box-shadow, ...
/* ❌ 브라우저: Layout(재계산) → Paint(재그리기) → Composite(합성) 반복 */
```

**해결:**
```css
/* GPU 가속 속성만 애니메이션 */
transition: opacity 0.3s, transform 0.3s, background-color 0.3s;
/* 이 3개 속성만 변함:
   - opacity: Composite만 필요 (⚡ 가장 빠름)
   - transform: Composite만 필요 (⚡ 가장 빠름)
   - background-color: Paint + Composite (중간)
*/
```

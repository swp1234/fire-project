세션 종료 시 PROGRESS.md 자동 업데이트.

1. PROGRESS.md 읽기
2. 이번 세션에서 수행한 모든 작업 요약 (git diff --stat 참조)
3. 세션 기록 추가:
   - `### 세션N (날짜) - 한줄 요약`
   - 주요 작업 3~5개 bullet point
4. "다음 우선순위" 섹션: 완료 항목 제거, 새로 발견된 이슈 추가
5. 이전 세션 테이블에 현재 세션 한줄 추가, 오래된 세션 상세 제거
6. **총 140줄 이하** 유지 (초과 시 오래된 세션 압축)
7. 프로젝트 규모 수치 업데이트 (앱 수, 블로그 수 등)

규칙:
- **GA4/GSC 조회 절대 금지** — 완료 작업 요약만
- 간결하게: 한 작업당 1줄
- 미완료 항목은 "다음 우선순위"로 이동

# i18n 다국어 지원 가이드

> 모든 앱/게임은 반드시 12개 언어를 지원해야 함

## 지원 언어 (12개)

| 코드 | 언어 | 비고 |
|------|------|------|
| `ko` | 한국어 | 기본 |
| `en` | English | 글로벌 기본 |
| `zh` | 中文 | 중국어 간체 |
| `hi` | हिन्दी | 힌디어 |
| `ru` | Русский | 러시아어 |
| `ja` | 日本語 | 높은 RPM |
| `es` | Español | 5억+ 화자 |
| `pt` | Português | 브라질 |
| `id` | Bahasa Indonesia | 2.7억 |
| `tr` | Türkçe | 터키어 |
| `de` | Deutsch | 높은 RPM |
| `fr` | Français | 프랑스어 |

## IIFE 안전 패턴 (CRITICAL)

```javascript
// 반드시 try-catch로 감싸기 - loader 멈춤 방지
(async function initI18n() {
    try {
        await i18n.loadTranslations(i18n.currentLang);
        i18n.updateUI();
    } catch (e) {
        console.warn('i18n load failed:', e);
    } finally {
        // loader는 반드시 숨기기
        const loader = document.getElementById('app-loader');
        if (loader) {
            loader.classList.add('hidden');
            setTimeout(() => loader.remove(), 300);
        }
    }
})();
```

## 번역 품질 규칙

- 자연스러운 현지어 (일본어 です/ます, 독일어 Sie, 프랑스어 vous)
- 앱별 도메인 용어 정확 번역
- JSON 키 구조 모든 언어 동일
- 새 기능 시 12개 언어 JSON 모두 업데이트

## 인코딩 안전 규칙 (CRITICAL)

- locale JSON은 반드시 `UTF-8`로 유지한다.
- Windows/PowerShell에서 `node -e`, here-string, one-liner로 locale JSON을 대량 수정할 때는 비ASCII 문자열을 직접 커맨드 본문에 넣지 않는다.
- 다국어 bulk edit가 필요하면 다음 둘 중 하나만 사용한다.
  - 저장된 스크립트 파일에서 `UTF-8`로 읽고 쓰기
  - ASCII-safe payload(`\\uXXXX` escape 등)로 JSON 키만 갱신
- locale 수정 직후에는 최소 `ko`, `ja`, `zh`, `ru` 4개 언어를 다시 열어 `?`, `�`, 깨진 구두점이 없는지 확인한다.
- 배포 전 검증은 최소 다음 순서를 지킨다.
  1. locale JSON parse 확인
  2. `git diff --check`
  3. 로컬 렌더링 또는 DOM 텍스트 확인
  4. 라이브 locale JSON endpoint 확인
  5. 라이브 실제 페이지 렌더링 확인
- 번역 사고가 발생하면 핫픽스만으로 끝내지 말고, 같은 세션에 이 문서나 운영 문서에 재발 방지 규칙을 추가한다.

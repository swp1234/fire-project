# 🚀 Fire Project 모델 전환 가이드

이 문서는 프로젝트 효율성을 극대화하기 위한 4단계 모델 배정 및 전환 규칙을 정의합니다. 모든 작업 시 이 지침에 따라 최적의 모델을 선택하거나 전환을 제안합니다.

---

## 🏗️ 4단계 모델 배정표

| 모델 | 별명 | 핵심 역할 | 주요 추천 작업 |
| :--- | :--- | :--- | :--- |
| **Gemini Flash** | **워커** | 초안 대량생산 | 번역 초안, 블로그 초적안, 메타/FAQ/CTA 후보안, 대량 콘텐츠 포맷팅 |
| **Gemini 3.1 Pro** | **분석가** | 분석 / 리서치 | GA4/GSC 데이터 해석, 키워드 클러스터링, 경쟁사 벤치마크, 패턴 분석, UI 리뷰 |
| **Claude 4.6 Sonnet** | **실무자** | 실제 구현 / 수정 | 코드 수정(HTML/CSS/JS), i18n/json/locale 작업, SEO 실제 반영, 기능 개선 |
| **Claude 4.6 Opus** | **전략가** | 전략 / 고난도 판단 | 서비스 재설계, 수익화 전략, 브랜드 포지셔닝, 핵심 카피 최종본, 우선순위 설계 |

---

## 🛠️ 작업 유형별 상세 배정

### 1. 콘텐츠 생산 및 번역
- **Gemini Flash**: 12개 언어 번역 초안 작성, 블로그 개요 생성
- **Gemini 3.1 Pro**: 번역 품질 비교 검토 보조
- **Claude 4.6 Sonnet**: 파일을 직접 수정하여 블로그 및 메타 정보 실제 반영 (현재 작업 중)
- **Claude 4.6 Opus**: 주요 페이지의 최종 톤앤매너 전문 감수

### 2. 검색 엔진 최적화 (SEO)
- **Gemini Flash**: 메타 타이틀/디스크립션 후보 대량 생성
- **Gemini 3.1 Pro**: GSC 노출 키워드 패턴 및 카니발라이제이션 분석
- **Claude 4.6 Sonnet**: 내부 링크 삽입 및 스키마 마크업 정리
- **Claude 4.6 Opus**: 전체 SEO 로드맵 및 핵심 키워드 전략 수립

### 3. 개발 및 UX/UI
- **Gemini 3.1 Pro**: 스크린샷 리뷰 및 UI 비교 분석
- **Claude 4.6 Sonnet**: 실제 구현, 버그 수정, 반응형 레이아웃 개선
- **Claude 4.6 Opus**: 정보 구조(IA) 재설계 및 핵심 전환(Conversion) 동선 설계

---

## ⚖️ 빠른 선택 규칙
1. **많이 뽑아야 한다** (Fast & Cheap) → **Gemini Flash**
2. **비교/분석해야 한다** (Read & Analysis) → **Gemini 3.1 Pro**
3. **직접 고쳐야 한다** (Build & Fix) → **Claude 4.6 Sonnet**
4. **결정해야 한다** (Strategy & Decision) → **Claude 4.6 Opus**
